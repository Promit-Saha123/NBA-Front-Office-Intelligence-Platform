# Data Audits

Results of auditing candidate and selected data sources, produced by
[workflows/audit-data-source.md](../../workflows/audit-data-source.md).

Each audit documents field coverage, season coverage, identifier stability,
traded-player representation, data-quality findings, and licensing verification for
one source.

## Completed Audits

* [FiveThirtyEight NBA RAPTOR — narrative audit](fivethirtyeight-raptor-audit.md)
  (2026-07-19) — the primary audit document; outcome: permitted for pipeline
  prototyping only. Audited under the then-approved BPM target; also unusable for
  PCE construction (no box-score fields), see
  [decision 0003](../decisions/0003-internal-player-impact-target.md).
  Automated profiles generated by `scripts/audit_data_source.py`:
  * [historical_RAPTOR_by_player.csv](fivethirtyeight-raptor-historical-by-player.md)
  * [historical_RAPTOR_by_team.csv](fivethirtyeight-raptor-historical-by-team.md)
  * [modern_RAPTOR_by_player.csv](fivethirtyeight-raptor-modern-by-player.md)
* [FiveThirtyEight NBA Elo — narrative audit](fivethirtyeight-nba-elo-audit.md)
  (2026-07-20) — team game outcomes 1947–2015, CC BY 4.0; permitted prototyping
  component for the PCE-era team-outcome side (no player stats, no possessions).
  Automated profile: [nbaallelo.csv](fivethirtyeight-nba-elo-games.md).

Candidates rejected on licensing before data retrieval (Kaggle Basketball-Reference
mirror, stats.nba.com, `nba-player-advanced-metrics`, Sports Reference custom
licensing) are documented in
[data-source-evaluation.md §5](../data-source-evaluation.md) and
[decision 0001](../decisions/0001-historical-data-source.md).
